setDrips

  • setDrips(erc20Address: string, currentReceivers: A.DripsReceiverStruct[], balanceDelta: BigNumberish, newReceivers: DripsReceiver[]): Promise<ContractTransaction>
    • Defined in src/AddressAppClient.ts:310

    Sets the client's signer drips configuration.

    throws

    DripsErrors.invalidDripsReceiver if any of the new receivers is not valid.

    Parameters

    • erc20Address: string

      The ERC20 token (address) to use.

    • currentReceivers: A.DripsReceiverStruct[]

      The signer's drips receivers that were set in the last drips configuration update. If this is the first update, pass an empty array. The default value is an empty array.

    • balanceDelta: BigNumberish

      The drips balance change to be applied. Positive to add funds to the drips balance, negative to remove them. The default value is 0.

    • newReceivers: DripsReceiver[]

      The new signer's drips receivers.

      Any "falsy" config.start or config.duration property, for any receiver, will be set to 0. This means "use the timestamp when drips are configured" and " drip until the balance runs out", respectively. See also DripsReceiver.

      Duplicate receivers are not allowed. Any duplicate entry will be removed. The default value is an empty array.

    Returns Promise<ContractTransaction>

    A Promise which resolves to the contract transaction.

Static create

  • create(provider: JsonRpcProvider): Promise<AddressAppClient>
    • Defined in src/AddressAppClient.ts:93

    Creates a new AddressAppClient instance.

    throws

    DripsErrors.invalidArgument if the provider argument has a "falsy" value, or the provider is connected to an unsupported chain.

    throws

    DripsErrors.invalidAddress if the provider's signer address is not valid.

    Parameters

    • provider: JsonRpcProvider

      The provider.

      The provider needs to have a signer associated with it that will be used to sign transactions.

      This signer will be the "user" the new instance will act on behalf of.

      The provider can connect to the following supported networks:

      • 'mainnet': chain ID 1
      • 'rinkeby': chain ID 4
      • 'goerli': chain ID 5
      • 'matic': chain ID 137
      • 'mumbai': chain ID 80001

    Returns Promise<AddressAppClient>

    A Promise which resolves to the new AddressAppClient instance.
